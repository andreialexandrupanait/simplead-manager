# Impeccable Design Context

## Design Context

### Users
WordPress agency owners and operators who manage multiple client websites. Desktop-primary usage during work hours in an office/home-office setting. They need to monitor site health, manage backups/updates/security, and generate client reports efficiently. Technical users who value density and control over decoration.

### Brand Personality
**Professional, reliable, capable.** The interface should evoke a sense of **control and confidence** — users should feel they have everything under control and can clearly see the state of all their sites at a glance.

### Aesthetic Direction
**Evolve the current direction incrementally.** Keep the purple accent (`#8D5CF5`) + dark navy sidebar (`#1A1A2E`) foundation, but refine it toward the quality bar set by Stripe Dashboard, Linear.app, and Notion:
- Clean, fast, professional — calm under pressure
- Strong typography with clear hierarchy
- **Light-mode first, dark-mode parity** (not just "inverted colors")
- Focus on data clarity and information density

**Anti-references (must NOT look like):**
- Generic WordPress admin plugins (cluttered, dated)
- AI-slop SaaS dashboards: cyan-on-dark, purple→blue gradients, gradient text, glassmorphism, glow effects
- Side-stripe borders on cards/alerts (`border-left: 4px solid <color>`)
- Hero metric template (big number + label + sparkline decoration)
- Identical card grids repeated endlessly

### Design Principles
1. **Clarity over decoration** — Every element should communicate, not decorate. Remove visual noise that doesn't serve the user's task.
2. **Confidence through data** — Present site health, uptime, and status in ways that give immediate understanding. Color-code meaningfully, not decoratively.
3. **Density with breathing room** — Pack information efficiently (agency users manage many sites) but use spacing hierarchy to prevent cognitive overload.
4. **Consistency compounds trust** — Uniform patterns, predictable interactions, and a cohesive token system build subconscious trust in the tool.
5. **Adapt, don't amputate** — Mobile and responsive views should reorganize content, not hide critical functionality.

### Technical Stack
- Laravel 11 + Livewire 4 + Blade + Tailwind CSS 3 + Alpine.js
- Chart.js for data visualization, TipTap for rich text editing
- Vite for asset bundling
- Class-based dark mode with server-side persistence

### Accessibility Requirements
- WCAG AA minimum
- Color-independent status indicators (not color-only)
- Keyboard navigable
- Reduced motion support needed
